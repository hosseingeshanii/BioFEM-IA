Detailed Description

Distributed geometry kernel for Loop subdivision surfaces using PETSc DMPlex.

This module implements a parallel geometry computation pipeline for triangulated immersed-boundary surfaces discretized with Loop subdivision stencils. The mesh topology is distributed across MPI ranks with PETSc's DMPlex infrastructure; patch stencils that cross partition boundaries are serviced by a PetscSF coordinate broadcast set up once at initialization.

Terminology

  • Original index: the vertex or cell index as it appears in the serial IBMNodes arrays (x_bp, y_bp, z_bp, nv1/nv2/nv3, patch, …). These indices are stable across the full run and are stored in DMLabels so they survive distribution.
  • DM local point index: the index used inside a distributed DM's chart [pStart, pEnd). Cells, edges, and vertices share one flat namespace. These indices are local to each rank and bear no relation to original indices without consulting the label.
  • Star Forest (PetscSF): PETSc's abstraction for one-to-many MPI patterns. A graph of roots (authoritative data on the owning rank) and leaves (remote copies). PetscSFBcast sends root → leaves; PetscSFReduce accumulates leaves → root. The SF built here has nroots = ibm->n_v (the full original vertex array) and nleaves = nRemoteNodes (cross-rank vertices needed by this rank's patch stencils).

Distribute the serial DMPlex and return per-rank original-index arrays.

Builds a serial (uninterpolated) DMPlex on rank 0, distributes it with overlap=0, and extracts per-rank origCell[] / origVert[] arrays directly from the migration Star Forest. No DMLabels are used: label stratification was the previous O(N^2) bottleneck.

After this call every rank holds:

  • A distributed DM (no ghost cells; overlap=0)
  • origCell[lc]: original IBMNodes cell index for local cell lc, or -1
  • origVert[lv]: original IBMNodes vertex index for local vertex lv, or -1
Parameters

[out]dmDistDistributed DM (caller must DMDestroy).
[out]origCellOutPer-rank original cell index array (size cEnd-cStart).
[out]origVertOutPer-rank original vertex index array (size vEnd-vStart).

Build the DMPlexPatchExchange SF for cross-rank coordinate fetching.

Each cell's Loop subdivision stencil references up to 16 vertices by original index. Some of those vertices are owned by other MPI ranks. This function determines exactly which remote vertices this rank needs, deduplicates them, and builds a PetscSF so that UpdatePatchRemoteCoordinates_() can broadcast updated coordinates each geometry iteration in O(nRemoteNodes) communication.

Complexity note: A naive implementation would check locality by scanning all local vertices for each patch node query (O(N^2/P) total). Instead, two boolean lookup arrays indexed by original vertex ID are built in one O(N/P) walk before the loop:

  • isLocalOrig[n_v]: true iff original vertex i is a non-ghost vertex on this rank. Replaces the former per-query linear scan.
  • alreadyAdded[n_v]: deduplication flag; set when a remote node is registered, replacing the former O(nRemoteNodes) linear scan. The main loop therefore runs in O(nLocalCells * max_patch_size).

SF layout:

  • nroots = ibm->n_v (full original vertex array; each rank is a potential root)
  • nleaves = nRemoteNodes (vertices this rank fetches from others)
  • remote[i] = {ownerRank[orig_node[i]], orig_node[i]} (root address)
Parameters
[in]layoutLocal patch layout (provides dm, nLocalCells, patch array).
[in]ibmIBM mesh (n_v, coordinate arrays used as SF root data).
[in]origVertorigVert[lv]: original IBMNodes vertex index for local vertex lv.
[in]ownerRankGlobal owner table from BuildOriginalVertexOwnerTable_().
[out]exchangeInitialised exchange struct (caller must call DMPlexPatchExchangeDestroy_ when done).

Populates ctx->ibm_to_global_dof0[v]: the first global DOF index of vertex v in any MPI Vec (e.g. Fint). Must be called after InitVecs() so that Vec ownership ranges are known. Result is identical on all ranks (built via MPI_Allreduce MAX).

Used by FInternalAct() to route VecSetValues calls to the correct global indices when assembling off-process force contributions.

Parameters
femFE structure with initialised geom_ctx and created Vecs.

)

Drop-in geometry call: setup + update + destroy in one shot.

Wraps FEM_DMPlexGeomSetup / FEM_DMPlexGeomUpdate / FEM_DMPlexGeomDestroy for call sites that do not persist the context across iterations. The comm argument controls parallelism: PETSC_COMM_SELF = serial (rank-0 only), PETSC_COMM_WORLD = fully distributed.

Deprecated:
Prefer storing the context in FE and calling Setup once at init, then Update each iteration. This function rebuilds the DM and SF on every call which is expensive for many timesteps.
